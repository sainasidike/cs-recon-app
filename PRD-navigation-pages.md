# PRD: CS 智能对账 PC 端 - 多页面导航系统

## 1. 背景与问题

当前侧边栏有三个导航入口（对账工作台、报告中心、文档管理），但点击后都停留在同一个页面。用户无法通过导航切换到不同功能区，导航栏形同虚设。

## 2. 产品目标

将 PC 端从「单页工具」升级为「多功能工作台」，三个导航页各自承载独立职责：
- **对账工作台**：发起新对账、查看进行中/最近的对账项目
- **报告中心**：汇总所有已完成的对账报告，支持查看、导出、对比
- **文档管理**：管理所有上传过的源文件，支持预览、复用、删除

## 4. 各页面详细设计

### 4.1 对账工作台（现有，微调）

**保持不变的部分：**
- 统计卡片行
- 上传区 + Demo 快捷按钮
- 最近对账列表（卡片网格）

**需要修改的部分：**
- 当 `navPage !== 'workspace'` 时不渲染此视图
- 「进行中」状态卡片增加「继续」按钮，点击可恢复对账流程

**布局：**

```
│  报告中心                                        │
├────────┬────────┬────────┬────────────────────────┤
│ 总报告  │ 本月   │ 平衡率  │ 已导出                  │

├────────┴────────┴────────┴────────────────────────┤
│ [全部] [已平衡] [未平衡]       🔍 搜索报告...       │
├───────────────────────────────────────────────────┤
│ 项目名称          │ 场景    │ 匹配率 │ 状态  │ 时间    │ 操作  │
│ 银行对账-4/28     │ 银行对账 │ 84.2% │ ✓平衡 │ 04-28  │ 👁📥🗑 │
│ 费用报销-4/25     │ 费用报销 │ 91.7% │ ✓平衡 │ 04-25  │ 👁📥🗑 │

```

**数据来源：**
- 从 `projectsHook` 获取所有 `status === 'completed'` 的项目
- 通过 `getProjectResult(projectId)` 获取报告详情（匹配率、平衡状态等）
- `resultSummary` 已经存储了 `isBalanced`、`matchRate`、`completedAt`

**交互：**
- 点击「查看」→ 调用 `onOpenProject(project)` 进入 ProjectDetailPage
- 点击「导出Excel」→ 调用现有 `exportToExcel` 工具
- 点击「导出PDF」→ 调用浏览器打印 / 生成 PDF
- 点击「删除」→ 二次确认后调用 `deleteProject(id)`
- 筛选标签切换「全部/已平衡/未平衡」

**空状态：**
- 图标 + "暂无对账报告"
- 描述："完成对账流程后，报告将自动保存在这里"
- CTA 按钮："去工作台开始对账"

**数据来源：**
- 遍历所有项目的 `files` 元信息（已存在于 localStorage 的 project meta 中）
- 文件 blob 存储在 IndexedDB 中（通过 `getProjectFiles(projectId)` 获取）

**交互：**
- 点击文件名 → 左右分栏预览（复用 SplitDocPreview 组件）
- 「复用」按钮 → 将文件加入当前对账的文件列表，跳回工作台
- 「删除」→ 二次确认后删除（如果文件仅属于一个项目则一并清理）
- 类型筛选标签
- 文件名搜索

**空状态：**
- 图标 + "暂无文档"
- 描述："上传的对账文件将自动归档在这里，方便随时复用"
- CTA 按钮："上传文件"

## 6. 需要新增的 Hook 方法

在 `useProjects` hook 中补充：

```javascript
// 获取所有文件的扁平列表（跨项目）
getAllFiles() → [{ fileName, fileSize, fileType, projectId, projectName, uploadedAt }]

// 获取已完成项目（报告中心用）
getCompletedProjects() → projects.filter(p => p.status === 'completed' || p.status === 'archived')

// 按文件类型统计
getFileStats() → { total, excel, pdf, image }
```

## 7. 技术实现方案

### 7.1 路由切换

在 `HomePage.jsx` 中根据 `navPage` prop 决定渲染内容：

```jsx
// HomePage.jsx
if (navPage === 'reports') return <ReportsView ... />;
if (navPage === 'docs') return <DocsView ... />;
// 默认 workspace
return <WorkspaceView ... />;
```

### 7.2 新增组件

```
src/pages/HomePage.jsx
├── WorkspaceView (现有逻辑，提取为子组件)
├── ReportsView (新增)
└── DocsView (新增)
```

不需要创建新页面文件，全部在 HomePage 内部作为子视图实现，通过 `navPage` 切换。原因：
- 共享同一个 sidebar layout
- 共享 projectsHook 数据
- 避免路由复杂度

### 7.3 样式方案

在 `components.css` 中新增：
- `.reports-view` 报告中心容器
- `.reports-table` 报告表格
- `.reports-stat-row` 统计摘要行
- `.docs-view` 文档管理容器
- `.docs-table` 文件表格
- `.docs-stat-row` 统计摘要行
- `.empty-state` 通用空状态组件
- `.table-action-btn` 表格操作按钮

## 8. 交互细节

| 场景 | 行为 |
|------|------|
| 侧边栏点「对账工作台」 | 切换到 workspace 视图 |
| 侧边栏点「报告中心」 | 切换到 reports 视图 |
| 侧边栏点「文档管理」 | 切换到 docs 视图 |
| 报告中心点「查看」 | 调用 onOpenProject，进入 ProjectDetailPage |
| 文档管理点文件名 | 在当前页面展开左右分栏预览 |
| 文档管理点「复用」 | 文件加入 parsedFiles，navPage 切回 workspace |
| 任何场景快捷入口 | loadDemo + navPage 切回 workspace |
| 新建对账按钮 | reset + navPage 切回 workspace |
| 对账流程进行中 | sidebar 隐藏，不受 navPage 影响 |

## 9. 空状态设计

所有页面均需空状态处理：

- **工作台空状态**（现有）：📋 暂无对账记录 + "上传文件或选择 Demo 开始第一次对账"
- **报告中心空状态**：📊 暂无对账报告 + "完成对账流程后，报告将自动保存在这里" + [去工作台开始对账]
- **文档管理空状态**：📁 暂无文档 + "上传的对账文件将自动归档在这里" + [上传文件]
